## Design direction (before coding)

Commit to a clear aesthetic before writing any code. Define:

- **Purpose**: what problem does this solve and who uses it.
- **Tone**: pick one and execute it fully — brutalist, minimal, maximalist,
  retro-futuristic, editorial, luxury, playful, organic, industrial, art deco.
- **Differentiation**: the one visual decision that makes this unforgettable.
- **Constraints**: framework, performance, accessibility requirements.

## Implementation rules

Produce working, production-grade code. Match implementation complexity to the
aesthetic vision: maximalist designs require extensive animation and layering;
minimalist designs require precision in spacing, type, and restraint.

## Aesthetics

**Typography**: avoid Arial, Inter, Roboto, and system fonts. Choose characterful
display fonts paired with refined body fonts.

**Color**: commit to a dominant palette with sharp accents. Avoid evenly
distributed or timid palettes. Use CSS variables for consistency.

**Motion**: prefer CSS-only animations for HTML. Use Motion library for React.
Prioritize high-impact moments: staggered page load reveals and purposeful hover
states over scattered micro-interactions.

**Layout**: use asymmetry, overlap, diagonal flow, and grid-breaking elements.
Alternate between generous negative space and controlled density.

**Backgrounds**: avoid plain solid colors. Use gradient meshes, noise textures,
geometric patterns, layered transparencies, dramatic shadows, or grain overlays
to create atmosphere and depth.

## Hard prohibitions

- No purple gradients on white backgrounds.
- No Space Grotesk, Inter, Roboto, or Arial.
- No predictable card-grid layouts without deliberate subversion.
- No design that could have been generated without reading the context.
- No two designs that converge on the same aesthetic choices.
